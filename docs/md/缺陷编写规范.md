# 缺陷编写规范

## 引言

### 目的

规范和统一缺陷提交流程，缺陷内容需要包含清晰重现步骤、错误截图、错误日志、接口请求内容和预期结果。

## 缺陷提交规范

:::info 缺陷内容

【步骤】 1. 操作的网站地址、用户名和密码 2. 进入目标页面的操作路径 3. 具体做了什么操作触发缺陷 4. 需要用到前置数据的，测试步骤需要尽可能描述清楚 【结果】 文字描述+报错截图+错误日志 【预期结果】 有对应的原型页面，最好贴上需求内容

:::
